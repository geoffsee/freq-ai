### Step 1 — Find or create the persona synthesis issue

Run:
```
gh issue list --state open --label "persona-synthesis" --json number,title --limit 5
```

- If an open persona-synthesis issue already exists, **edit it in place** with
  `gh issue edit <number> --body-file -` (or `--title` if the headline changed). Reuse
  the same issue so the persona synthesis remains a single living document.
- If none exists, create one with:
  ```
  gh issue create \
    --title "Persona Synthesis: <YYYY-MM-DD> — <one-line headline>" \
    --body "<full persona set with all cards and the Persona Landscape summary, plus a footer: 'Generated by Persona Synthesis agent run on <YYYY-MM-DD>.'>" \
    --label "persona-synthesis"
  ```

Use today's date for `<YYYY-MM-DD>`. The title headline should name the primary persona
or capture the key insight about the user landscape.

### Step 2 — Echo the issue URL

After creating or editing the issue, print the issue URL so it appears in the editor
panel output. Format: `Persona synthesis published: <URL>`

Do NOT write any files to disk — the GitHub issue IS the artifact.
