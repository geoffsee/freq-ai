## Instructions

Produce at least 15 distinct UX/UI ideas across these buckets:

### Interaction Patterns
Ideas for how users interact with the system — navigation flows, input methods, gesture
patterns, keyboard shortcuts, command palettes, progressive disclosure, inline editing,
drag-and-drop, undo/redo patterns, or novel interaction paradigms. Focus on reducing
clicks, cognitive load, and time-to-task-completion.

### Information Architecture
Ideas for how content and features are organized — menu structures, page hierarchies,
search and filtering, breadcrumbs, contextual help, onboarding sequences, empty states,
content grouping, or dashboard layouts. Focus on findability, mental model alignment,
and information scent.

### Visual & Motion Design
Ideas for how the interface looks and feels — color systems, typography scales, spacing
rhythms, iconography, illustration styles, micro-animations, loading transitions, skeleton
screens, dark/light mode, data visualization, or brand expression. Focus on clarity,
delight, and visual hierarchy.

### Accessibility & Inclusion
Ideas for making the experience work for everyone — screen reader improvements, keyboard
navigation enhancements, color contrast fixes, reduced motion alternatives, internationalization,
right-to-left support, cognitive accessibility, plain language, or assistive technology
integration. Focus on removing barriers and expanding who can use the product effectively.

## Format

For each idea: one-sentence description, one-sentence rationale explaining the user benefit.
No sizing, no commitment, no ranking. Do **not** create GitHub issues.
Do **not** filter or evaluate ideas — the human will react in feedback.

This is a DRAFT for human review. The human will keep some ideas, drop others,
expand on a few, and provide feedback before anything is finalised.
