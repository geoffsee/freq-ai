### Step 1 — Find or create the journey map issue

Run:
```
gh issue list --state open --label "journey-map" --json number,title --limit 5
```

- If an open journey-map issue already exists, **edit it in place** with
  `gh issue edit <number> --body-file -` (or `--title` if the headline changed). Reuse
  the same issue so the journey maps remain a single living document.
- If none exists, create one with:
  ```
  gh issue create \
    --title "Journey Maps: <YYYY-MM-DD> — <one-line headline>" \
    --body "<full journey maps with all stages, emotional arcs, and Cross-Journey Analysis, plus a footer: 'Generated by Journey Mapping agent run on <YYYY-MM-DD>.'>" \
    --label "journey-map"
  ```

Use today's date for `<YYYY-MM-DD>`. The title headline should capture the key journey
insight or the most critical pain point discovered.

### Step 2 — Echo the issue URL

After creating or editing the issue, print the issue URL so it appears in the editor
panel output. Format: `Journey maps published: <URL>`

Do NOT write any files to disk — the GitHub issue IS the artifact.
