### Step 1 — Find or create the usability audit issue

Run:
```
gh issue list --state open --label "usability-audit" --json number,title --limit 5
```

- If an open usability-audit issue already exists, **edit it in place** with
  `gh issue edit <number> --body-file -` (or `--title` if the headline changed). Reuse
  the same issue so the usability audit remains a single living document.
- If none exists, create one with:
  ```
  gh issue create \
    --title "Usability Audit: <YYYY-MM-DD> — <one-line headline>" \
    --body "<full audit with all 10 heuristic evaluations, scorecard, critical failures, quick wins, and strengths, plus a footer: 'Generated by Usability Audit agent run on <YYYY-MM-DD>.'>" \
    --label "usability-audit"
  ```

Use today's date for `<YYYY-MM-DD>`. The title headline should capture the most critical
usability finding. The body must contain the complete final audit.

### Step 2 — Echo the issue URL

After creating or editing the issue, print the issue URL so it appears in the editor
panel output. Format: `Usability audit published: <URL>`

Do NOT write any files to disk — the GitHub issue IS the artifact.
