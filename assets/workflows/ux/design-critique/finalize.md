### Step 1 — Find or create the design critique issue

Run:
```
gh issue list --state open --label "design-critique" --json number,title --limit 5
```

- If an open design-critique issue already exists, **edit it in place** with
  `gh issue edit <number> --body-file -` (or `--title` if the headline changed). Reuse
  the same issue so the design critique remains a single living document.
- If none exists, create one with:
  ```
  gh issue create \
    --title "Design Critique: <YYYY-MM-DD> — <one-line headline>" \
    --body "<full critique with all 8 dimensions, scorecard, design wins, design debts, and consistency matrix, plus a footer: 'Generated by Design Critique agent run on <YYYY-MM-DD>.'>" \
    --label "design-critique"
  ```

Use today's date for `<YYYY-MM-DD>`. The title headline should capture the most important
design finding. The body must contain the complete final critique.

### Step 2 — Echo the issue URL

After creating or editing the issue, print the issue URL so it appears in the editor
panel output. Format: `Design critique published: <URL>`

Do NOT write any files to disk — the GitHub issue IS the artifact.
