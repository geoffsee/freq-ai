### Step 1 — Find or create the accessibility review issue

Run:
```
gh issue list --state open --label "accessibility-review" --json number,title --limit 5
```

- If an open accessibility-review issue already exists, **edit it in place** with
  `gh issue edit <number> --body-file -` (or `--title` if the headline changed). Reuse
  the same issue so the accessibility review remains a single living document.
- If none exists, create one with:
  ```
  gh issue create \
    --title "Accessibility Review: <YYYY-MM-DD> — <one-line headline>" \
    --body "<full review with all WCAG 2.1 criteria evaluations, compliance scorecard, critical failures, compliance gaps, quick wins, and testing gaps, plus a footer: 'Generated by Accessibility Review agent run on <YYYY-MM-DD>.'>" \
    --label "accessibility-review"
  ```

Use today's date for `<YYYY-MM-DD>`. The title headline should capture the overall
compliance status or the most critical finding. The body must contain the complete
final review.

### Step 2 — Echo the issue URL

After creating or editing the issue, print the issue URL so it appears in the editor
panel output. Format: `Accessibility review published: <URL>`

Do NOT write any files to disk — the GitHub issue IS the artifact.
