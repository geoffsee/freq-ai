## Instructions

Incorporate the feedback above. Expand dimensions the human flagged for deeper
investigation. Drop or compress dimensions they deprioritised. Challenge findings
they questioned. Add new threads they surfaced.

Then produce the FINAL research output with:

1. **Research Dimensions** — Each dimension with its findings in the structured format
   (finding summary, signal strength, source, evidence, contradictions, trajectory).
   Only include dimensions the human kept or expanded.

2. **Cross-Cutting Analysis** — Updated contradiction map, signal convergence, blind
   spots, temporal dynamics, adversarial read, and adjacent possible.

3. **Research Digest** — The dense 10-15 bullet summary of highest-signal findings,
   updated to reflect feedback. Each bullet tagged with source dimension(s) and
   confidence level. This digest is the primary artifact consumed by downstream
   workflows.

4. **Research Questions for Next Cycle** — 3-5 open questions that this research
   surfaced but could not resolve. These seed the next deep-research run.

## Publishing the Research as a GitHub Issue

After completing the final research, publish it as a GitHub issue so it is
reviewable, durable, and consumable by downstream workflows (ideation, UXR Synth,
strategic review).

### Step 1 — Capture the list of prior open deep-research issues

Run:
```
gh issue list --label deep-research --state open --json number --jq '.[].number'
```

Save the list of issue numbers — you'll close them in Step 3 with a back-reference.

### Step 2 — Create the new deep-research issue

Run:
```
gh issue create \
  --title "Deep Research: <YYYY-MM-DD> — <one-line headline of highest-signal finding>" \
  --body "<full research output including all dimensions, cross-cutting analysis, research digest, and next-cycle questions, plus a footer: 'Generated by Deep Research agent run on <YYYY-MM-DD>.'>" \
  --label "deep-research"
```

Use today's date for `<YYYY-MM-DD>`. The title headline should capture the single
most important finding. The body must contain the complete final research output.
Capture the new issue number (`#<new>`) from the URL output.

### Step 3 — Close prior deep-research issues with a back-reference

For each issue number captured in Step 1, close it with a single comment that already
includes the new issue number — no follow-up edit needed:
```
gh issue close <NUMBER> --comment "Superseded by #<new>."
```

### Step 4 — Echo the issue URL

After creating the issue, print the issue URL so it appears in the editor panel output.
Format: `Deep research published: <URL>`

Do NOT write any files to disk — the GitHub issue IS the artifact.
